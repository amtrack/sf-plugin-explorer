FAQ

    How does this compare to the awesome-sfdx-plugins list? The awesome-sfdx-plugins by Shane McLaughlin provides a community-driven list of sf plugins with a hand-crafted description. The SF Plugin Explorer scrapes data from various sources to allow searching for sf plugins including their commmands. Both the awesome-sfdx-plugins and the SF Plugin Explorer are great resources to discover new and useful sf plugins.
    How often is the data updated? The data is updated daily using a scheduled GitHub Action. Last updated:
    Why is plugin xyz not listed? We search the NPM registry for the keywords sf-plugin and sfdx-plugin which should be included in any plugin generated using the sf plugins generate command. Please double-check that your plugin includes these keywords in package.json > keywords. If you're not the author of a plugin you're missing here, please consider creating a Pull Request like this to help them improve their discoverability. The npm search result is then processed to exclude plugins pointing to an archived or forked GitHub repository. Please check the build logs and search for your plugin name to see if it is excluded.
    Is this open-source? Yes, both the website and the scripts to query the data are available at https://github.com/amtrack/sf-plugin-explorer. Please note that it is still in a very early stage so please ping us before making contributions.
